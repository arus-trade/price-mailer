# Сервис рассылки прайс-листов — arus-trade.ru

---

## Описание

Автоматический сервис, который каждый день в **06:00 (МСК)**:

1. Подключается к почтовому ящику `price@arus-trade.ru` по IMAP
2. Находит последнее письмо с темой **«Прайс-лист»**
3. Скачивает Excel-вложение (файл вида `Прайс-лист 17.03.2026.xlsx`)
4. Пересылает его с адреса `noreply@arus-trade.ru` получателям:
   - `prosecco.msk@mail.ru`
   - `sale@arus-trade.ru`
   - `bronitorg1@proton.me`

Почта работает через **Mail.ru для бизнеса**. Дополнительные библиотеки не требуются — скрипт использует только стандартную библиотеку Python.

## Установка

### Вариант А — автоматическая (рекомендуется)

```bash
# 1. Скопируйте папку на сервер
scp -r price_mailer/ user@ваш-сервер:/tmp/

# 2. На сервере запустите установку
cd /tmp/price_mailer
chmod +x setup.sh
./setup.sh
```

Скрипт установит Python (если нужно), скопирует файлы в `/opt/price_mailer/` и добавит задание в cron на 06:00 МСК.

### Вариант Б — ручная установка

**1. Скопируйте файлы** `price_mailer.py` и `config.json` на сервер, например в `/opt/price_mailer/`.

**2. Убедитесь, что Python установлен:**

```bash
python3 --version    # нужен 3.8+
```

**3. Добавьте задание в cron:**

```bash
crontab -e
```

Если часовой пояс сервера UTC (по умолчанию):
```
0 3 * * * cd /opt/price_mailer && /usr/bin/python3 price_mailer.py >> /opt/price_mailer/logs/cron.log 2>&1
```

Если часовой пояс сервера уже МСК:
```
0 6 * * * cd /opt/price_mailer && /usr/bin/python3 price_mailer.py >> /opt/price_mailer/logs/cron.log 2>&1
```

Узнать часовой пояс: `timedatectl`

## Настройка паролей

Перед запуском нужно создать **пароли приложений** для двух почтовых ящиков в Mail.ru для бизнеса (обычные пароли от аккаунтов не подойдут).

**1.** Войдите в панель администратора: https://biz.mail.ru/

**2.** Для ящика `price@arus-trade.ru`:
   - Настройки ящика → Безопасность → Пароли приложений
   - Создайте пароль для «Почтовых клиентов»
   - Скопируйте пароль

**3.** Повторите то же самое для `noreply@arus-trade.ru`

**4.** Убедитесь, что **IMAP** включён для `price@` и **SMTP** включён для `noreply@`

**5.** Откройте `config.json` и вставьте пароли:

```json
{
    "imap": {
        "password": "сюда_пароль_для_price"
    },
    "smtp": {
        "password": "сюда_пароль_для_noreply"
    }
}
```

## Настройка config.json

| Параметр | Описание | Значение по умолчанию |
|----------|----------|-----------------------|
| `imap.server` | IMAP-сервер для получения писем | `imap.mail.ru` |
| `imap.port` | Порт IMAP (SSL) | `993` |
| `imap.login` | Ящик, куда приходят прайсы | `price@arus-trade.ru` |
| `imap.password` | Пароль приложения | — нужно заполнить — |
| `smtp.server` | SMTP-сервер для отправки | `smtp.mail.ru` |
| `smtp.port` | Порт SMTP (SSL) | `465` |
| `smtp.login` | Ящик-отправитель | `noreply@arus-trade.ru` |
| `smtp.password` | Пароль приложения | — нужно заполнить — |
| `search_subject` | Тема письма для поиска | `Прайс-лист` |
| `recipients` | Список получателей | 3 адреса |
| `email_subject` | Тема исходящего письма | `Актуальный прайс-лист` |
| `email_body` | Текст исходящего письма | Стандартное приветствие |

Чтобы добавить или убрать получателя — отредактируйте массив `recipients` в config.json.

## Устранение проблем

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `IMAP4.error: LOGIN failed` | Неверный пароль для `price@` | Пересоздайте пароль приложения |
| `IMAP4.error: LOGIN failed` | IMAP отключён | Включите IMAP в настройках ящика |
| `SMTPAuthenticationError` | Неверный пароль для `noreply@` | Пересоздайте пароль приложения |
| «Письма с такой темой не найдены» | Нет писем с темой «Прайс-лист» | Проверьте ящик `price@` вручную |
| «Вложение не найдено» | Письмо есть, но без Excel-файла | Убедитесь, что вложение называется `Прайс-лист...xlsx` |
| Письма не доходят | Попали в спам | Проверьте спам у получателей |
| Cron не срабатывает | Неверное время или путь | Проверьте `crontab -l` и часовой пояс |
| Cron работает, но ошибка | Нет Python по пути | Проверьте `which python3` и обновите путь в cron |
